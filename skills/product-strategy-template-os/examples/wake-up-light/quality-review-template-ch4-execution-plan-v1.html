必须继承的第 3 章问题

用户第一眼到底看见了什么？

我们要验证用户会不会把光效、卧室对象感、简单控制和品质可信看成更高价值。如果用户只看成闹钟或声音机，那么产品会被低价锚拖住。

执行循环

正式正文开始后，每个节点都按这个循环推进，通过后再进入下一个节点。

1. Input

确认当前节点需要的真实输入：评论、Q&A、视频、报告、问卷、体验洞察。没有就标缺。

2. Acquire

能 MCP 拉就拉；需要浏览器就浏览器；需要人工就写成待补输入，不用想象替代。

3. Processing

按模板方法处理：文本分析、场景矩阵、用户旅程、需求权重、创意线索。

4. Supplement Review

AI 先自查缺证：是否评论数量不够、是否只看正评、是否缺 Q&A、是否场景视频不足。

5. Red-team

从公司能力和消费者前台视角反驳结论：这到底是差异价值，还是普通功能包装？

6. Output

只输出本节点该输出的内容，不跨到产品规划。完整 HTML 作为最终人类判断页面。
